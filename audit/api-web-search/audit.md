# 大模型 API 联网能力闭环审计

审计日期：2026-07-16

## 实施结果

已按本审计完成闭环改造：新增服务商/协议/模型能力矩阵、独立“测试联网”、供应商专用请求映射、统一搜索事件、结构化引用/查询/工具错误持久化、报告联网状态与审计检查，以及多供应商自动化测试。未知 Chat Completions 接口不再被默认视为支持联网。

方舟/OpenAI/xAI Responses、Anthropic Messages、Gemini Grounding、Perplexity Sonar、OpenRouter、智谱、Moonshot 与阿里百炼已分别映射。最终是否可用仍取决于用户所选模型、账号权限、额度、浏览器域名权限和服务商 CORS，因此设置页要求通过“测试联网”取得结构化引用后再视为已验证。

## 原始核查问题（改造前）

1. 配置入口：联网开关能够按连接保存，界面可见。
2. 能力识别：当前主要按 API 协议判断，而不是按服务商、模型与协议三者共同判断。
3. 请求映射：方舟 Responses、OpenRouter、智谱、Moonshot 等有映射；Anthropic 与 Gemini 原生协议未附加联网工具，通用 Chat Completions 分支会向其他服务商发送泛化的 `web_search` 工具。
4. 连接测试：只验证基础模型调用，不验证联网工具是否真正执行。
5. 流式解析：只归一化文本和 Token 用量，没有归一化搜索工具事件、查询、结构化引用和工具错误。
6. 来源入库：报告来源主要依赖模型正文中的 Markdown 链接或裸 URL，而不是供应商返回的结构化 citations/search results。
7. 最终综合：综合请求明确关闭联网，无法在最终结论阶段独立核验时效性信息或冲突证据。
8. 测试：现有相关测试 13 项通过；其中只有方舟 Responses 覆盖了联网请求映射，没有覆盖多供应商引用与真实搜索闭环。

## 已补齐项

- 服务商 + 协议 + 模型能力矩阵与安全降级。
- 独立“测试联网”，必须取得搜索事件或结构化引用。
- 统一流事件与 citations、search results、grounding metadata、HTTP 200 工具错误解析。
- 研究任务、Agent 结果和报告持久化实际搜索状态、查询、引用和错误。
- 报告界面区分未联网、已请求但未验证、实际联网已验证。
- 多服务商请求映射、响应解析、编排持久化与审计测试。

## 证据限制

本次未使用用户真实 API Key，也未发起付费服务商调用；结论来自代码路径、现有自动化测试、本地 UI 检查及各服务商官方接口规范核对。
