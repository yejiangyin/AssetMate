| A04 | P1 高 | “删除持仓”在内部被当成按现价卖出 | 删除错误数据会生成卖出事件和已平仓收益，污染历史收益 | 明确拆分“卖出/清仓”和“删除错误记录”；删除不得生成交易或收益 |
| A05 | P1 高 | 基础币种没有在全产品统一生效 | 持仓页使用用户币种，但总览和收益页大量固定为 CNY，切换币种后页面口径互相矛盾 | 所有金额统一走货币上下文和换算服务，禁止页面局部硬编码 CNY |
| A06 | P1 高 | 设置页“数据不会上传任何服务器”的表述不真实 | 投研会把研究上下文发送给模型 API，外部搜索也会发送主题/标的；勾选持仓分析时还可能发送私有持仓 | 改为“默认本地保存；启用 AI/搜索时会发送本次所需字段”，并在运行前提供发送字段摘要 |
| A07 | P1 高 | 报告“审计通过”规则对含数字表格的报告几乎不可达 | 只要报告含数值表格，“抽样数据点”固定为警告；总体通过又要求全部检查为通过，导致高质量数据报告也无法通过 | 区分阻断、警告、信息项；警告不应自动导致总体失败 |

| A09 | P1 高 | “独立来源”实际只统计 URL 数量 | 同一网站的多个链接也会被认为是多个独立来源，来源多样性被高估 | 按注册域名/发布机构去重，并区分原始资料、媒体转述、模型生成链接 |
| A10 | P1 高 | 收益页的期间结果与收益排行口径冲突 | 实机周度结果及构成均为 ¥0，但排行仍展示累计持仓浮盈；用户会误以为这些是周度贡献 | 基准快照不足时隐藏期间排行，或明确标注“自建仓累计”，不得混入期间收益 |
| A11 | P1 高 | 自动分红税未进入持仓级总收益 | 现金分红计入持仓累计分红，但代扣税只进入全局事件；持仓卡片总收益高于总览/收益页 | 税费事件需要可追溯到持仓，并进入持仓级统一损益计算 |
| A12 | P1 高 | 首次买入和定投税费未进入持仓级总收益 | 买入/DCA 费用只进入全局事件，持仓卡片不扣除，跨页面收益不一致 | 所有成本统一入账到持仓账本；页面只读取同一个收益计算结果 |
| A13 | P1 高 | A 股送股/转增被统一标记为“拆股” | 数量和成本数学上可能接近，但通知、历史事件和报告语义错误 | 增加“送股”“转增”事件类型，保留来源字段，不要复用拆股文案 |

## 四、跨模块一致性结论

目前不是所有闭环都使用同一数据源：

1. **收益闭环未统一**：总览/收益分析偏向事件账，持仓卡片偏向持仓局部字段，已实现收益又把平仓摘要和事件相加，产生三套口径。
2. **设置闭环未统一**：配置存储、密钥存储、会话内存和 IndexedDB 分散，缺少跨窗口同步和统一备份恢复。
3. **投研闭环基本成形但可信度层仍弱**：搜索、模型、报告和导出已串联；“独立来源”和“审计通过”的定义仍会给用户过高信心。
4. **公司行动闭环部分成形**：事件识别、通知、持仓调整存在，但送转语义、失败反馈、持仓级税费归集仍不完整。
5. **币种闭环未完成**：基础币种不是全局单一事实源，总览、持仓和收益页面会出现不同币种口径。

## 五、建议修复顺序

1. 先修 A01、A02，恢复回测主链路并消除不可逆数据丢失风险。
2. 建立“唯一收益账本”，一起解决 A03、A04、A10、A11、A12、A18～A20。
3. 统一基础币种和金额格式服务，解决 A05。
4. 修正研究审计单位、独立来源、随机抽样与措辞，解决 A07～A09、A23、A26。
5. 补齐跨窗口设置同步、会话密钥策略和研究备份恢复，解决 A24、A28、A29。
6. 完善公司行动事件类型、批量失败反馈、通知逐条管理，解决 A13～A17。
7. 最后统一字号、危险操作样式、来源标识和行情加载反馈。

## 六、必须新增的自动化测试

1. 从投研中心进入回测，填写参数并成功启动的浏览器测试。
2. 清理数据前后的存储清单测试，断言提示文案与实际删除键完全一致。
3. 完整平仓 + 历史分红 + 卖出税费场景，断言已实现收益只计算一次。
4. 删除持仓不生成卖出事件；清仓才生成卖出事件。
5. 买入、DCA、现金分红代扣税在持仓卡片、总览、收益页三处金额一致。
6. 切换 USD/HKD/CNY 后，总览、持仓、收益、报告币种一致。
7. 英文 K/M/B/T 数值解析和研究审计测试。
8. 同域多个 URL 不得通过“独立来源”检查。
9. 弹窗和侧边栏同时打开时，切换当前模型/搜索配置应实时同步。
10. 研究库达到上限前提示、完整导出、清理后恢复测试。
11. 公司行动单标的失败不影响其余标的，并能在通知中展示失败摘要。
12. 快照不足时，收益页不得把累计浮盈展示为期间贡献。

## 七、总体判断

当前版本可以继续作为开发测试版使用，但不建议在修复 A01～A13 前把它定义为“完整闭环”：基础工程质量不错，模型/搜索/研究和公司行动的框架也已搭好；主要风险集中在 **财务口径可信度、危险数据操作、跨页面一致性和事实审计表述**。这些问题不是纯 UI 瑕疵，会直接影响收益结论、历史数据和用户信任。
