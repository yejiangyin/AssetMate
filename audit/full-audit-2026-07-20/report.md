## 本次确认并修复的问题

1. **方舟 Agent Plan 联网能力判断错误**：模型 API Key 不再被误认为自动启用原生 `web_search`；Agent Plan 与独立方舟联网搜索现在分开配置、分开验证。
2. **方舟外部搜索缺少正式适配**：新增 SearchInfinity Bearer 鉴权、请求字段、时间范围、50 条单次上限、大小写字段及嵌套响应解析，并加入固定域名权限。
3. **原生联网工具被服务商拒绝后可能直接中断**：自动模式现在会记录失败事件，并无缝回退为不带原生搜索工具的模型请求；若配置了外部搜索则继续使用外部证据。
4. **联网状态存在误导**：执行模型、地址或必要 Key 未就绪时，不再显示“模型原生联网已启用”，而是明确提示先完成模型配置。
5. **外部搜索配置项与真实能力不一致**：方舟/Brave 下不再展示会被静默忽略的域名过滤；方舟自动摘要、Brave 额外摘要、Tavily/Exa 正文增强和自定义字段映射分别展示真实语义。
6. **免 Key 本地模型无法启动研究**：Ollama、LM Studio 等 `authMode=none` 连接不再被错误拦截；运行前改为校验实际路由模型、地址和按鉴权方式决定是否需要 Key。
7. **本地日期可能跨时区偏移一天**：行情日期、公司行动默认日期和报告 ZIP 日期统一使用本地日历日期。
8. **长同步错误挤压顶部布局**：总览、持仓、收益统一压缩为“同步异常”，完整错误保留在悬浮提示和无障碍名称中。
9. **无障碍名称缺失**：总览隐私按钮、设置页三个开关、回测最终清仓开关、报告导出/删除按钮均补齐语义和状态。
10. **方舟错误与响应兼容不足**：补充 `ResponseMetadata.Error.Message` 和 `Data/Results` 等嵌套结构解析，避免服务正常返回却被判定为无结果。

## 联网闭环说明

研究运行时按以下顺序处理：

1. 插件预取结构化行情与财务数据，并记录服务商、实际数据日期、新鲜度和失败原因。
2. 若当前协议、服务商和模型具备可验证的原生搜索能力，发送原生搜索工具请求。
3. 若原生工具不支持或被拒绝，自动模式会继续运行；已配置独立搜索服务时注入带 URL 的外部证据，否则使用预取数据并在报告中披露限制。
4. 只有收到实际搜索事件或结构化引用才标记“原生联网已验证”，不会因为模型声称自己联网而误判。
5. 独立搜索的“单次结果数/整次保留来源”只约束插件调用的搜索 API，不限制模型服务商自己的搜索次数和引用数量。

## 非阻断增强建议

1. **真实凭证烟雾测试**：当前不能替用户使用未提供的模型/搜索 Key 发起付费请求。建议在设置中保存“最近成功测试时间、模型、协议、联网证据类型”，便于长期诊断供应商变更。
2. **区域化基本面覆盖**：美股 SEC/公司财务链路较完整；A 股、港股、基金的分析师预测和细分财务数据仍依赖可选数据源。可增加交易所公告/基金定期报告的结构化适配。
3. **搜索证据质量策略**：可在未来增加来源域名信誉等级、发布日期冲突提示和同一事实的跨域交叉验证分数；这应作为证据质量层，不应简单提高来源数量上限。
